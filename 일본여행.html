이 플래너 사용법

✏️ 텍스트 직접 편집 브라우저에서 바로

상단 배너의 편집 모드 켜기 버튼을 누르면 장소명, 메모, 시간, 소제목 등 모든 텍스트를 클릭해서 바로 수정할 수 있습니다.

단, 이 방식은 새로고침하면 초기화됩니다. 인쇄하거나 스크린샷으로 남기는 용도, 또는 발표·공유 직전 빠른 수정에 적합합니다.

💡
지도 마커 위치나 날짜 구조를 바꾸려면 아래 TRIP 데이터 수정 방법을 사용하세요.

🗂️ TRIP 데이터 수정 영구 반영

HTML 파일을 텍스트 에디터(VS Code, 메모장 등)로 열면 파일 상단에 const TRIP = { ... } 블록이 있습니다. 이 블록만 수정하면 지도 마커, 타임라인, 텍스트, 페리 배너가 모두 자동으로 반영됩니다.

// 장소 하나의 구조 예시 { time: "14:00", // 시간 표시 name: "감천문화마을", // 장소명 note: "골목길 뷰포인트", // 메모 tag: "문화·역사", // 태그 (아래 4종류) lat: 35.0974, // 위도 (지도 마커) lng: 129.0106, // 경도 (지도 마커) }
🏷️
사용 가능한 태그 4종류: 자연·힐링  /  문화·역사  /  맛집·카페  /  이동·페리

➕ 장소 추가하기

stops 배열 안에 아래 형식으로 객체를 복사해서 붙여넣으면 됩니다. 좌표는 Google 지도에서 원하는 위치를 우클릭 → 좌표 복사로 얻을 수 있습니다.

  1. Google 지도에서 원하는 장소를 검색합니다.
  2. 장소를 우클릭하면 좌표(위도, 경도)가 맨 위에 표시됩니다. 클릭하면 복사됩니다.
  3. HTML 파일의 해당 날짜 stops: [ ] 배열에 붙여넣습니다.
  4. 저장 후 브라우저에서 새로고침하면 마커가 지도에 자동 추가됩니다.
GitHub Pages

배포하기

🚀 GitHub Pages 무료 배포

이 파일 하나만 업로드하면 됩니다. 외부 의존성은 Google Fonts와 Leaflet.js CDN뿐이므로 별도 빌드 과정이 없습니다.

  1. GitHub 저장소 생성 — 새 Public 저장소를 만듭니다.
  2. 파일 업로드 — travel-itinerary.html을 파일명 index.html로 올립니다.
  3. Pages 설정 — Settings → Pages → Source를 main 브랜치 / / (root)로 설정합니다.
  4. 배포 완료 — 잠시 후 https://[유저명].github.io/[저장소명]으로 접속 가능합니다.
🔗
저장소를 Private으로 만들어도 GitHub Pages는 Public URL로 공개됩니다. 비공개가 필요하다면 Netlify 또는 Vercel을 이용하세요 — 둘 다 HTML 파일 하나를 드래그&드롭으로 배포할 수 있습니다.
Claude에게 수정 요청

AI로 내용 업데이트하기

💬 수정된 파일을 Claude에게 주기

내용을 직접 수정한 HTML 파일을 Claude에게 첨부하면, 구조를 유지하면서 원하는 변경사항을 반영한 새 파일로 만들어 드립니다.

  1. HTML 파일을 텍스트 에디터로 열어 TRIP 데이터 또는 텍스트를 수정합니다.
  2. 수정된 파일을 저장합니다.
  3. Claude 대화창에 파일을 첨부하고 원하는 변경 내용을 설명합니다.
  4. Claude가 수정된 파일을 다시 생성해 드립니다.
